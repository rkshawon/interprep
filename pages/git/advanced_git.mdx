# 1. Git Internals

Git has 3 major areas:

| Area              | Meaning              |
| ----------------- | -------------------- |
| Working Directory | Current file changes |
| Staging Area      | Prepared changes     |
| Repository        | Saved commit history |

Workflow:

```text
Edit files
   ↓
git add .
   ↓
git commit
   ↓
git push
```

Git commits are snapshots.

Each commit has:

- unique hash
- parent commit
- metadata
- file snapshot

Example commit hash:

```text
9c12abf2
```

# 3. Rebase vs Merge

## Merge

```bash
git merge feature-auth
```

Creates merge commit.

History preserved.

Example:

```text
A---B---C main
     \   \
      D---E feature
```

---

## Rebase

```bash
git rebase main
```

Moves feature commits on top of latest branch.

Cleaner history.

Example:

```text
A---B---C main
         \
          D'---E' feature
```

---

## When to Use

| Use Case               | Recommended |
| ---------------------- | ----------- |
| Shared branch          | merge       |
| Local cleanup          | rebase      |
| Clean history          | rebase      |
| Preserve exact history | merge       |

# 6. Reset vs Revert

## Reset

Moves branch pointer backward.

### Soft Reset

```bash
git reset --soft HEAD~1
```

Removes commit.
Keeps changes staged.

---

### Mixed Reset

```bash
git reset HEAD~1
```

Removes commit.
Keeps file changes unstaged.

---

### Hard Reset

```bash
git reset --hard HEAD~1
```

Deletes:

- commit
- staging
- working changes

⚠️ Dangerous.

---

## Revert

Safer for shared branches.

```bash
git revert commit-hash
```

Creates NEW commit that undoes changes.

# 7. Detached HEAD

Occurs when checking out specific commit.

```bash
git checkout 9c12abf2
```

You are no longer on a branch.

Fix:

```bash
git checkout main
```

Or create branch:

```bash
git checkout -b recovery-branch
```

# 8. Interactive Rebase

Very powerful cleanup tool.

```bash
git rebase -i HEAD~5
```

Allows:

- squash commits
- rename messages
- remove commits
- reorder commits

---

## Common Options

| Command | Meaning         |
| ------- | --------------- |
| pick    | keep commit     |
| squash  | combine commits |
| reword  | edit message    |
| drop    | delete commit   |

---

# 9. Git Reflog

Shows ALL reference history.

```bash
git reflog
```

Very useful for recovering lost commits.

Example:

```bash
git reset --hard HEAD@{2}
```

---

# 10. Squash Commits

Combine multiple commits into one.

```bash
git rebase -i HEAD~3
```

Change:

```text
pick
pick
pick
```

To:

```text
pick
squash
squash
```

Useful before pull request.

# 15. Git Bisect

Binary search for bad commit.

```bash
git bisect start
```

Mark bad:

```bash
git bisect bad
```

Mark good:

```bash
git bisect good commit-hash
```

Git automatically narrows problem commit.

---

# 16. Git Worktree

Multiple working directories from same repo.

```bash
git worktree add ../hotfix hotfix-branch
```

Useful for:

- parallel development
- avoiding branch switching

---

# 17. Git Submodules

Nested repositories.

Add submodule:

```bash
git submodule add <repo-url>
```

Clone with submodules:

```bash
git clone --recurse-submodules <repo>
```

# 18. Large File Handling

Git is NOT optimized for huge binaries.

Use Git LFS.

Install:

```bash
git lfs install
```

Track files:

```bash
git lfs track "*.zip"
```

# 21. Production Best Practices

## Recommended Rules

- never force push shared branches
- commit small logical changes
- use meaningful commit messages
- pull before push
- rebase feature branches frequently
- squash noisy commits before merge
- protect main branch
- use pull requests

---

# Recommended Professional Workflow

```bash
# Update main branch
git checkout main
git pull

# Create feature branch
git checkout -b feature-payment

# Work and commit
git add .
git commit -m "added stripe payment"

# Rebase latest main
git fetch origin
git rebase origin/main

# Push branch
git push -u origin feature-payment
```
