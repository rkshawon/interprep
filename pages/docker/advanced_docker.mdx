# 1. Docker Architecture

Docker has several major components:

| Component     | Purpose                |
| ------------- | ---------------------- |
| Docker Engine | Main runtime           |
| Docker CLI    | Command line interface |
| Docker Daemon | Background service     |
| Images        | Blueprints/templates   |

# 3. Layers and Caching

Docker images are layered.

Example:

```Dockerfile
FROM node:20
COPY package*.json ./
RUN npm install
COPY . .
```

Each instruction creates a layer.

---

## Why Order Matters

This is optimized:

```Dockerfile
COPY package*.json ./
RUN npm install
COPY . .
```

Because dependency layer gets cached.

---

## Bad Example

```Dockerfile
COPY . .
RUN npm install
```

Every code change invalidates cache.

# 7. Volumes and Persistent Storage

---


---


# 9. Multi-Stage Builds

Very important for production.

## Example

```Dockerfile
# Build stage
FROM node:20 AS builder

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

# Production stage
FROM node:20-alpine

WORKDIR /app

COPY --from=builder /app/dist ./dist
COPY package*.json ./

RUN npm install --only=production

CMD ["node", "dist/index.js"]
```

---

## Benefits

- smaller image
- faster deployment
- more secure
- no dev dependencies

# 10. Docker Compose

Manage multi-container applications.

```yaml
    build: .
```

## Detached Mode

```bash

```

---

## Stop Compose


# 18. Security Best Practices

## Use Smaller Images

Prefer:

```Dockerfile
node:20-alpine
```

---

## Avoid Root User

```Dockerfile
USER node
```

---

## Do Not Store Secrets in Dockerfile

Bad:

```Dockerfile
ENV DB_PASSWORD=secret
```

Use:

- env files
- secrets manager

---

## Scan Vulnerabilities

```bash
docker scan myapp
```

# 19. Production Dockerfile Optimization

## Development Dockerfile

```Dockerfile
FROM node:20

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

CMD ["npm", "start"]
```

---

## Production Dockerfile

```Dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

RUN npm run build

CMD ["node", "dist/index.js"]
```

---

## Why Better

- smaller image
- faster startup
- deterministic install
- no dev dependencies

# 20. Cleanup and Maintenance

## Remove Unused Images

---

## Remove Everything Unused


## Remove Volumes Too

```bash

```

# 22. Production Deployment Workflow

## Recommended Workflow

```bash
# Build image
docker build -t myapp:v1 .

# Run container
docker run -d \
  --name backend \
  --restart unless-stopped \
  -p 8000:8000 \
  --env-file .env \
  myapp:v1
```

- use multi-stage builds

- use health checks
- use smaller base images

- pin versions
- use volumes for databases
- separate development and production configs
