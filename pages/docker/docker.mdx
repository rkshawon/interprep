# Docker Common Commands

## 1. Build Docker Image

Build image from Dockerfile.

```bash
docker build -t myapp .
```

### Explanation

- `build` → create image
- `-t` → tag/name
- `myapp` → image name
- `.` → current folder

---

# 2. Run Container

Run image as container.

```bash
docker run myapp
```

Run with port mapping:

```bash
docker run -p 8000:8000 myapp
```

Run with environment file:

```bash
docker run -p 8000:8000 --env-file .env myapp
```

---

# 3. Run Container in Background

Detached mode.

```bash
docker run -d myapp
```

With ports + env:

```bash
docker run -d -p 8000:8000 --env-file .env myapp
```

---

# 4. Name a Container

```bash
docker run --name backend myapp
```

Example:

```bash
docker run -d -p 8000:8000 --env-file .env --name backend myapp
```

---

# 5. List Running Containers

```bash
docker ps
```

---

# 6. List All Containers

Including stopped containers.

```bash
docker ps -a
```

---

# 7. Stop Container

```bash
docker stop <container-id>
```

Example:

```bash
docker stop backend
```

---

# 8. Restart Container

```bash
docker restart backend
```

---

# 9. Remove Container

```bash
docker rm <container-id>
```

Example:

```bash
docker rm backend
```

---

# 10. List Images

```bash
docker images
```

---

# 11. Remove Image

```bash
docker rmi myapp
```

---

# 12. View Logs

```bash
docker logs <container-id>
```

Live logs:

```bash
docker logs -f backend
```

---

# 13. Enter Container Shell

Using sh:

```bash
docker exec -it backend sh
```

Using bash:

```bash
docker exec -it backend bash
```

---

# 14. Remove All Stopped Containers

```bash
docker container prune
```

---

# 15. Remove Unused Docker Cache

```bash
docker system prune
```

⚠️ Removes unused:

- containers
- images
- cache
- networks

---

# Most Common Real-World Flow

```bash
# Build image
docker build -t app .

# Run app
docker run -d -p 8000:8000 --env-file .env --name backend app

# Check running containers
docker ps

# See logs
docker logs -f backend

# Stop container
docker stop backend

# Remove container
docker rm backend
```

# Important Docker Concepts

## Port Mapping

```bash
docker run -p HOST_PORT:CONTAINER_PORT app
```

Example:

```bash
docker run -p 8000:8000 app
```

Meaning:

- Left side = your PC port
- Right side = container app port

---

## Environment Variables

Pass env file:

```bash
docker run --env-file .env app
```

Docker does NOT automatically load `.env`.

---

## Docker Image vs Container

| Type      | Meaning            |
| --------- | ------------------ |
| Image     | Blueprint/template |
| Container | Running instance   |

---

## Dockerfile Base Image

Example:

```Dockerfile
FROM node:20
```

Meaning:

- use official Node.js image
- includes Linux + Node.js + npm
- your app builds on top of it
