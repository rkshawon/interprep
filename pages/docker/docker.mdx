# Docker Common Commands

## 1. Build an Image

Build a Docker image from the current directory.

```bash
docker build -t app .
```

### Explanation

- `build` → Build an image from a Dockerfile
- `-t app` → Assign the image the name `app`
- `.` → Use the current directory as the build context

---

# 2. Run a Container

Run an image as a container.

```bash
docker run app
```

Run with port mapping:

```bash
docker run -p 8000:8000 app
```

Run with environment variables:

```bash
docker run -p 8000:8000 --env-file .env app
```

---

# 3. Run in Detached Mode

Run the container in the background.

```bash
docker run -d app
```

Example:

```bash
docker run -d -p 8000:8000 --env-file .env app
```

---

# 4. Name a Container

```bash
docker run --name backend app
```

Example:

```bash
docker run -d \
  -p 8000:8000 \
  --env-file .env \
  --name backend \
  app
```

---

# 5. Run an Interactive Container

Useful for debugging.

```bash
docker run -it app sh
```

or

```bash
docker run -it app bash
```

---

# 6. Automatically Remove Container After Exit

```bash
docker run --rm app
```

---

# 7. List Running Containers

```bash
docker ps
```

---

# 8. List All Containers

Including stopped containers.

```bash
docker ps -a
```

---

# 9. Stop a Container

```bash
docker stop backend
```

---

# 10. Start a Stopped Container

```bash
docker start backend
```

---

# 11. Restart a Container

```bash
docker restart backend
```

---

# 12. Remove a Container

```bash
docker rm backend
```

Force removal:

```bash
docker rm -f backend
```

---

# 13. List Images

```bash
docker images
```

---

# 14. Remove an Image

```bash
docker rmi app
```

Force removal:

```bash
docker rmi -f app
```

---

# 15. View Container Logs

```bash
docker logs backend
```

Follow logs in real time:

```bash
docker logs -f backend
```

Show the last 100 lines:

```bash
docker logs --tail 100 backend
```

---

# 16. Execute Commands Inside a Running Container

Using `sh`:

```bash
docker exec -it backend sh
```

Using `bash`:

```bash
docker exec -it backend bash
```

---

# 17. Inspect a Container

```bash
docker inspect backend
```

---

# 18. View Resource Usage

```bash
docker stats
```

---

# 19. Remove Stopped Containers

```bash

```

---

# 20. Remove Unused Docker Resources

```bash
docker system prune
```

Remove everything including unused images:

```bash
docker system prune -a
```

⚠️ This removes unused:

- Containers
- Networks
- Images
- Build cache

---

# 23. View Docker Networks


# Common Development Workflow

```bash
# Build the image
docker build -t app .

# Run the container
docker run -d \
  -p 8000:8000 \
  --env-file .env \
  --name backend \
  app

# Verify it's running
docker ps

# View logs
docker logs -f backend

# Enter the container
docker exec -it backend sh

# Restart after changes
docker restart backend

# Stop the container
docker stop backend

# Remove the container
docker rm backend
```

# Important Docker Concepts

## Docker Image vs Container

| Image               | Container                    |
| ------------------- | ---------------------------- |
| Read-only blueprint | Running instance of an image |

---

## Port Mapping

```bash
docker run -p HOST_PORT:CONTAINER_PORT app
```

Example:

```bash
docker run -p 8000:8000 app
```

Meaning:

- **Host Port** → Port exposed on your computer
- **Container Port** → Port your application listens on inside the container

---

## Environment Variables

Pass variables from a file:

```bash
docker run --env-file .env app
```

> Docker **does not automatically load** your `.env` file.

---

## Mount a Local Folder (Volumes)

Useful during development.

```bash
docker run -v $(pwd):/app app
```

Windows PowerShell:

```powershell
docker run -v ${PWD}:/app app
```

---

## Dockerfile Base Image

```dockerfile
FROM node:20
```

This means:

- Start from the official Node.js 20 image
- Includes Linux, Node.js, and npm
- Your application is built on top of this image
