# JavaScript `try...catch` — Beginner to Advanced 🚀

## What is Exception Handling?

Imagine you're writing code that reads a file, calls an API, or parses JSON.

Sometimes things go wrong.

Instead of crashing your application, JavaScript allows you to **catch** the error and decide what to do.

This process is called **Exception Handling**.

```text
Without try...catch

Program
   │
   ▼
Error ❌
   │
Program Stops
```

```text
With try...catch

Program
   │
   ▼
try
   │
   ├── Success ✅
   │      │
   │      ▼
   │ Continue
   │
   └── Error ❌
          │
          ▼
       catch
          │
          ▼
 Continue Running
```

---

# Why do we need try...catch?

Without error handling:

```javascript
const user = JSON.parse("{");
console.log(user);
```

Output

```text
SyntaxError: Unexpected end of JSON input
```

Everything stops.

With try...catch:

```javascript
try {
  const user = JSON.parse("{");
  console.log(user);
} catch (error) {
  console.log("Invalid JSON");
}

console.log("Application still running");
```

Output

```text
Invalid JSON
Application still running
```

# finally

Sometimes we want code to execute whether an error happens or not.

```javascript
try {
  console.log("Start");
} catch (error) {
  console.log(error);
} finally {
  console.log("Always executes");
}
```

Output

```text
Start
Always executes
```

# throw

You can create your own errors.

```javascript
function divide(a, b) {
  if (b === 0) {
    throw new Error("Cannot divide by zero");
  }

  return a / b;
}
```

Usage

```javascript
try {
  divide(10, 0);
} catch (error) {
  console.log(error.message);
}
```

Output

```text
Cannot divide by zero
```

# Custom Error

```javascript
class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}
```

Usage

```javascript
function register(name) {
  if (!name) {
    throw new ValidationError("Name required");
  }
}
```

# Async try...catch

Most common interview question.

Without try...catch

```javascript
async function getUsers() {
  const response = await fetch("/users");

  const users = await response.json();

  return users;
}
```

If fetch fails

❌ Unhandled Promise Rejection

---

Correct way

```javascript
async function getUsers() {
  try {
    const response = await fetch("/users");

    const users = await response.json();

    return users;
  } catch (error) {
    console.log(error.message);

    return [];
  }
}
```

---

# Promise vs try...catch

Using Promise

```javascript
fetch("/users")
  .then((res) => res.json())
  .catch((error) => console.log(error));
```

Using async/await

```javascript
try {
  const users = await fetch("/users");
} catch (error) {
  console.log(error);
}
```

Interview preference:

Modern applications generally use **async/await + try...catch** because it is easier to read.

# What try...catch CANNOT catch

## Syntax errors during parsing

```javascript
if (
```

The program never starts.

---

## Errors in another asynchronous callback

```javascript
try {
  setTimeout(() => {
    throw new Error("Boom");
  }, 1000);
} catch (error) {
  console.log(error);
}
```

This will NOT work.

Correct

```javascript
setTimeout(() => {
  try {
    throw new Error("Boom");
  } catch (error) {
    console.log(error);
  }
}, 1000);
```

# Bad Practices ❌

## Empty catch

```javascript
try {
} catch {}
```

Never do this.

---

## Swallowing errors

```javascript
try {
  doSomething();
} catch {
  return;
}
```

Debugging becomes impossible.

---

## Using try...catch everywhere

Wrong

```javascript
try {
  console.log(name);
} catch {}
```

Use it only around operations that may genuinely fail.

---

# Best Practices ✅

✔ Catch only expected errors

✔ Log unexpected errors

✔ Create custom Error classes

✔ Don't ignore errors

✔ Use finally for cleanup

✔ Keep try blocks small

✔ Throw meaningful messages

# Interview Questions

## What is try...catch?

A mechanism to handle runtime exceptions without crashing the application.

---

## When does catch execute?

Only when an exception occurs inside the try block.

---

## Does finally always execute?

Yes.

Except when the process terminates immediately.

---

## Difference between throw and throws?

JavaScript has **throw**.

There is **no throws keyword**.

---

## Difference between throw new Error() and console.error()?

```javascript
throw new Error();
```

Stops execution.

```javascript
console.error();
```

Only logs.

---

## Can finally override return?

Yes.

```javascript
function demo() {
  try {
    return 1;
  } finally {
    return 2;
  }
}

console.log(demo());
```

Output

```text
2
```

Interviewers love this question.

---

## Why use custom errors?

Instead of

```javascript
throw new Error("Invalid");
```

Create

```javascript
throw new ValidationError("Email required");
```

Now you can distinguish error types.

---

## Can try...catch catch Promise rejection?

Yes, **only with await**.

```javascript
try {
  await fetch(url);
} catch (error) {
  console.log(error);
}
```

---

## Difference between Error and Exception?

JavaScript treats thrown `Error` objects as exceptions.

An **Error** is an object.

An **Exception** is the event of throwing that object.

# Advanced Interview Example

```typescript
class ApiError extends Error {
  constructor(
    message: string,
    public status: number,
  ) {
    super(message);
  }
}

async function getUser(id: number) {
  try {
    const response = await fetch(`/users/${id}`);

    if (!response.ok) {
      throw new ApiError("User not found", response.status);
    }

    return await response.json();
  } catch (error) {
    if (error instanceof ApiError) {
      console.log(error.status);
      console.log(error.message);
    } else {
      console.log("Unknown error");
    }
  } finally {
    console.log("Request finished");
  }
}
```

# Cheat Sheet

| Keyword      | Purpose               |
| ------------ | --------------------- |
| `try`        | Wrap risky code       |
| `catch`      | Handle errors         |
| `finally`    | Always executes       |
| `throw`      | Create an error       |
| `Error`      | Built-in error object |
| `instanceof` | Check error type      |

---

# Key Takeaways

- `try` contains risky code.
- `catch` runs only when an exception is thrown.
- `finally` executes whether an error occurs or not.
- Use `throw` to create meaningful custom errors.
- Prefer `async/await` with `try...catch` for asynchronous code.
- Never silently ignore errors.
- Keep `try` blocks focused and handle only expected failures.
- Understanding `try...catch`, `throw`, `finally`, custom errors, and async error handling is essential for JavaScript and TypeScript interviews.
