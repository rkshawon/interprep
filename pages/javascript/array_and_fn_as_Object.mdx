# JavaScript: Arrays & Functions as Objects - Complete Guide 🚀

JavaScript is a **prototype-based language**. Almost everything you work with is either an object or behaves like one.

Primitive values are **not objects**, while Arrays, Functions, Dates, Maps, Sets, and many other built-in types are specialized objects.

---

## JavaScript Types

| Type      | Example        | Object?                                                                      |
| --------- | -------------- | ---------------------------------------------------------------------------- |
| Number    | `42`           | ❌ Primitive                                                                 |
| String    | `"hello"`      | ❌ Primitive                                                                 |
| Boolean   | `true`         | ❌ Primitive                                                                 |
| BigInt    | `10n`          | ❌ Primitive                                                                 |
| Symbol    | `Symbol()`     | ❌ Primitive                                                                 |
| Undefined | `undefined`    | ❌ Primitive                                                                 |
| Null      | `null`         | ❌ Primitive _(although `typeof null === "object"` due to a historical bug)_ |
| Object    | `{}`           | ✅                                                                           |
| Array     | `[]`           | ✅                                                                           |
| Function  | `function(){}` | ✅                                                                           |

> **Remember:** Arrays and Functions are specialized kinds of objects.

# Arrays Are Objects

## Checking an Array's Type

```javascript
const arr = [1, 2, 3];

console.log(typeof arr);
// "object"

console.log(arr instanceof Array);
// true

console.log(arr instanceof Object);
// true

console.log(Array.isArray(arr));
// true (Recommended way)
```

### Why does `typeof` return `"object"`?

Arrays are objects whose internal prototype is `Array.prototype`.

```
arr
 │
 ▼
Array.prototype
 │
 ▼
Object.prototype
 │
 ▼
null
```

---

# Arrays Store Values Using Object Properties

Internally, array indexes are object property names.

```javascript
const arr = ["apple", "banana"];

console.log(arr[0]);
console.log(arr["0"]);

console.log(arr.length);
```

Output

```text
apple
apple
2
```

Even though we use numeric indexes, JavaScript converts them into string property names.

```javascript
const arr = ["A", "B"];

console.log(Object.keys(arr));
```

Output

```text
["0", "1"]
```

So internally it's similar to:

```javascript
{
  "0": "A",
  "1": "B"
}
```

with extra array behavior.

---

# Arrays Have a Special `length` Property

```javascript
const arr = [10, 20, 30];

console.log(arr.length);
```

Output

```text
3
```

Unlike normal objects, `length` is automatically managed.

```javascript
const arr = [];

arr.push(1);
arr.push(2);

console.log(arr.length);
```

Output

```text
2
```

Changing the length also changes the array.

```javascript
const arr = [1, 2, 3, 4];

arr.length = 2;

console.log(arr);
```

Output

```text
[1, 2]
```

---

# Arrays Can Have Custom Properties

Since arrays are objects, you can attach your own properties.

```javascript
const arr = [1, 2, 3];

arr.owner = "John";

console.log(arr.owner);
console.log(arr.length);
```

Output

```text
John
3
```

Notice that custom properties **do not** affect array indexing or `length`.

However, this is generally discouraged because it can make code confusing.

---

# Arrays Inherit from `Array.prototype`

Arrays automatically inherit useful methods.

```javascript
const arr = [1, 2, 3];

console.log(arr.map((x) => x * 2));
console.log(arr.filter((x) => x > 1));
console.log(arr.join("-"));
```

These methods come from `Array.prototype`.

You can verify this:

```javascript
const arr = [];

console.log(Object.getPrototypeOf(arr) === Array.prototype);
```

Output

```text
true
```

---

# Own Properties vs Inherited Properties

```javascript
const arr = [1, 2];

console.log(arr.hasOwnProperty("length"));
```

Output

```text
true
```

Why?

- `hasOwnProperty()` comes from `Object.prototype`
- `length` is an **own property** of the array instance

# Functions Are Objects

Functions are special objects that can be executed.

```javascript
function greet() {}

console.log(typeof greet);
```

Output

```text
function
```

Although `typeof` returns `"function"`, functions are still objects.

```javascript
console.log(greet instanceof Function);
console.log(greet instanceof Object);
```

Output

```text
true
true
```

---

# Functions Can Have Properties

```javascript
function greet() {
  console.log("Hello");
}

greet.language = "English";

console.log(greet.language);
```

Output

```text
English
```

Since functions are objects, you can attach properties to them.

---

# Functions Can Be Stored in Objects

```javascript
const user = {
  sayHi() {
    console.log("Hi!");
  },
};

user.sayHi();
```

Functions stored inside objects are called **methods**.

---

# Functions Have a `length` Property

Unlike arrays, a function's `length` means something different.

It tells you how many parameters are declared.

```javascript
function add(a, b, c) {}

console.log(add.length);
```

Output

```text
3
```

---

# Constructor Functions and `prototype`

Normal functions have a `prototype` property.

```javascript
function Person(name) {
  this.name = name;
}

console.log(Person.prototype);
```

You can add methods that every instance shares.

```javascript
Person.prototype.sayHello = function () {
  console.log(`Hello ${this.name}`);
};

const john = new Person("John");

john.sayHello();
```

Output

```text
Hello John
```

---

# Arrow Functions Don't Have a Prototype

```javascript
const greet = () => {};

console.log(greet.prototype);
```

Output

```text
undefined
```

Arrow functions cannot be used as constructors.

```javascript
new greet();
```

Output

```text
TypeError
```

# Arrays vs Objects

```javascript
const arr = ["A", "B"];

const obj = {
  0: "A",
  1: "B",
};

console.log(arr[0]);
console.log(obj[0]);
```

Both work similarly because array indexes are object property names.

The difference is that arrays provide:

- automatic `length`
- array methods
- iteration support
- optimized storage for indexed data

# Comparing Objects, Arrays, and Functions

| Feature                    | Object | Array | Function                           |
| -------------------------- | ------ | ----- | ---------------------------------- |
| Stores key-value pairs     | ✅     | ✅    | ✅                                 |
| Numeric indexing           | ❌     | ✅    | ❌                                 |
| Automatic `length`         | ❌     | ✅    | ❌ _(has parameter count instead)_ |
| Has methods via prototype  | ✅     | ✅    | ✅                                 |
| Callable                   | ❌     | ❌    | ✅                                 |
| Can have custom properties | ✅     | ✅    | ✅                                 |
| Iterable (`for...of`)      | ❌     | ✅    | ❌                                 |

---

# Key Takeaways

✅ Arrays are specialized objects.

✅ Array indexes are object property names stored as strings internally.

✅ Arrays automatically maintain a special `length` property.

✅ Arrays inherit methods like `map()`, `filter()`, and `reduce()` from `Array.prototype`.

✅ Functions are objects that can also be executed.

✅ Functions can store properties just like ordinary objects.

✅ Regular functions have a `prototype` property and can be used as constructors.

✅ Arrow functions do **not** have a `prototype` property and cannot be used with `new`.

✅ Everything in JavaScript's object system is connected through the **prototype chain**, enabling inheritance and shared behavior.
